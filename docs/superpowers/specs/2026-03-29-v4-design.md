# GuildMaster V4 — Design Spec

**Date:** 2026-03-28
**Branch:** v3 → v4
**Scope:** System infrastructure only. No new NPCs, cards, or arcs are created in this pass.

---

## Overview

V4 adds three interconnected systems:

1. **Content unlock system** — replaces the legacy trait system. Meta-progression unlocks new content (NPCs, arcs, events, cards) instead of granting numeric starting bonuses.
2. **Post-choice narrative** — a result text appears inline on every card (except rumors) after the player makes a choice, narratively encoding resource changes.
3. **Unlock toast** — an animated in-run notification when content is unlocked.

Additionally, the game start flow is redesigned with a tutorial card and explicit scenario selection.

---

## 1. Game Start Flow

The new start sequence replaces the current flow (which opened directly to guild naming):

```
Tutorial card (skippable)
        │
        ▼
Scenario selection
        │
        ▼
Guild naming
        │
        ▼
NPC selection
        │
        ▼
Run starts
```

### 1.1 Tutorial card

A single full-screen card shown before anything else. Content:

- **Who you are:** You are a guild master.
- **What to do:** Keep your guild running until the end of the scenario (60 cards). If any of your four resources — Gold, Adventurers, Quests, Equipment — hits 0 or 100, the run ends.
- **How:** Every turn, an NPC presents a situation. You choose a response. Every choice costs and gains resources.

The card has a checkbox: **"Don't show this again"**. When checked, `progress.skipTutorial` is set to `true` and persisted to localStorage. On subsequent runs, if `skipTutorial` is true, this screen is skipped entirely.

### 1.2 Scenario selection

A screen listing available arcs. Each arc shows:
- A short flavor description (1–2 sentences setting the narrative context)
- Its name

Only unlocked arcs are selectable. Locked arcs are shown greyed out with their unlock condition visible (matching the Discoveries menu style). Currently only **Bandit War** is available.

### 1.3 Guild naming

Flavor text updated to set the scene in roleplay tone:

> *"You've just arrived in a small town with big ambitions. A building stands ready to welcome adventurers — it just needs a name."*

Functionality unchanged.

### 1.4 NPC selection

Updated copy to hint at the recurrence and relationship system in roleplay tone:

> *"These faces will cross your path again and again. How you treat them shapes what they offer — and what they withhold."*

Only **unlocked** NPCs appear in the selection pool. Two of the four base NPCs are pre-unlocked from run 1; two are locked behind conditions (conditions defined when content is written).

## 2. Unlock Engine

### 2.1 Registry — `src/data/unlocks.js`

Pure data file. Each entry describes one piece of unlockable content:

```js
{
  id: 'npc-lord-farwick',        // unique identifier
  type: 'npc',                   // 'npc' | 'arc' | 'event' | 'card'
  name: 'Lord Farwick',
  emoji: '👑',
  flavor: 'A minor nobleman who trades favors like coin.',  // shown in menu when unlocked
  hint: 'A minor nobleman with powerful connections',       // shown in menu when locked (secret: false)
  secret: false,                 // if true, shown as '???' while locked (name and hint hidden)
  condition: {
    type: 'run-win',             // see condition types below
    resource: 'gold',            // optional, for resource-threshold and run-win+resource
    threshold: 80                // optional
  }
}
```

**Supported condition types:**

| `type` | Evaluated | Description |
|---|---|---|
| `arc-complete` | Run end | Player completed a specific arc. Requires `arc: 'arc-id'`. |
| `run-win` | Run end | Player won any run. |
| `run-loss` | Run end | Player lost any run. |
| `run-win` + `resource` + `threshold` | Run end | Player won with a resource above threshold at run end. |
| `flag` | After choice | A named flag is set in the current run. Requires `flag: 'flag-id'`. |
| `resource-threshold` | After choice | A resource crosses a threshold at any point in the run. Requires `resource` and `threshold`. |

New condition types can be added in future passes by extending the evaluator.

### 2.2 Engine — `src/engine/unlock-engine.js`

Responsibilities:
- Load the registry from `unlocks.js`
- Compare registry against `progress.unlockedContent[]` to determine what is still locked
- Evaluate conditions at two checkpoints (see below)
- When a condition is met: add to `progress.unlockedContent`, persist to localStorage, fire the toast callback

**Checkpoint 1 — after each choice** (`checkAfterChoice(gameState, flags)`):
Evaluates `flag` and `resource-threshold` conditions.

**Checkpoint 2 — on run end** (`checkOnRunEnd(outcome, finalState)`):
Evaluates `arc-complete`, `run-win`, `run-loss`, and `run-win`+resource conditions.
`outcome` is `{ result: 'win' | 'loss', arc: 'arc-id', finalResources: {...} }`.

**Storage:**
`progress.unlockedContent: string[]` — array of unlock `id` strings.

### 2.3 Progression changes

`src/engine/progression.js` is updated to:
- Remove legacy trait system (`activeLegacyTrait`, trait selection logic)
- Add `unlockedContent: []` to initial progress state
- Expose `isUnlocked(id)` and `getUnlockedByType(type)` helpers

`src/data/traits.js` is deleted.

The trait selection screen in `src/ui/ledger-view.js` is removed. The ledger shows the run summary and returns directly to the main menu.

## 3. Post-choice Narrative

### 3.1 Data

Each choice gains an optional `result` field (string). Cards without a `result` skip the narrative step silently. Rumors never have a `result` field.

```js
{
  label: 'Demand payment first',
  deltas: { gold: 20, adventurers: -15, quests: -8, equipment: 0 },
  result: 'Osric grits his teeth and pays. Half your crew rides out with him — and the board grows quieter.',
  // ... existing fields unchanged
}
```

**Writing rule:** Result text must narratively convey every resource that moves, its direction, and rough weight — without using numbers. Resources that don't move are not mentioned. The resource bar handles magnitude; the text handles meaning.

Examples:
- `gold: -15, quests: +20` → *"Osric can't pay much, but word spreads fast — the quest board fills before nightfall."*
- `adventurers: -10, equipment: +12` → *"The expedition costs two good fighters, but they return loaded with salvaged gear."*
- `gold: +20, adventurers: -15, quests: -8` → *"Osric grits his teeth and pays. Half your crew rides out with him — and the board grows quieter."*

This encoding future-proofs a "immersive" difficulty mode where delta numbers are hidden — the result text becomes the player's only resource feedback.

### 3.2 UI — `src/ui/card-view.js`

Flow after a choice is clicked:

1. Choice buttons fade out (~200ms)
2. Result text fades in below the situation text (same card, no navigation)
3. Resource bar animates simultaneously
4. After animation completes (~1500ms total), card slides out → next card slides in

No button, no extra screen, no interaction required.

## 4. Unlock Menu — "Discoveries"

### 4.1 Access

A **"Discoveries"** button is added to the main menu (between New Game and any existing secondary options). Opens `src/ui/unlock-menu.js` (new file).

### 4.2 Layout

Four collapsible category sections: **NPCs**, **Arcs**, **Events**, **Cards**.

Each entry renders in one of three states:

| State | Display |
|---|---|
| Unlocked | Emoji + name + `flavor` text |
| Locked, `secret: false` | 🔒 + name + condition text (greyed) |
| Locked, `secret: true` | 🔒 + "???" — no hint, no condition |

### 4.3 Placeholder state at launch

Until new content is added:

- **NPCs:** All 4 base NPCs listed. 2 pre-unlocked, 2 locked with conditions.
- **Arcs:** Bandit War listed (pre-unlocked).
- **Events:** Section shows *"More content coming soon."*
- **Cards:** Section shows *"More content coming soon."*

### 4.4 Data flow

`unlock-menu.js` reads `progress.unlockedContent[]` and compares against the full registry in `unlocks.js`. No game state dependency — pure read.

Navigation: back button returns to main menu. No animations required.

## 5. Unlock Toast

### 5.1 File

`src/ui/unlock-toast.js` (new file).

### 5.2 Animation sequence (~2500ms, non-blocking)

1. Toast slides in from top-right
2. Displays: 🔒 + content name + condition text
3. Lock icon animates: brief shake → breaks open
4. Background transitions: neutral dark → green
5. Text updates: 🔓 + content name + *"Unlocked"*
6. Toast fades out and slides away

### 5.3 Behavior

- **Non-blocking** — game flow does not pause during the toast
- **Queued** — if multiple unlocks fire simultaneously (e.g., at run end), toasts appear sequentially with a short delay between them
- **Suppressed** — toast does not fire if the player is on the main menu or Discoveries screen

### 5.4 Integration

`unlock-engine.js` accepts an optional `onUnlock(entry)` callback at init. `game.js` passes a callback that calls `unlock-toast.show(entry)`.

## 7. Out of Scope

- New NPC definitions (Lord Farwick unlock conditions, new characters)
- New arc definitions
- New event or card content
- Immersive difficulty mode (hiding delta numbers) — enabled by this design, built later
- Compound unlock conditions (AND/OR) — evaluator handles single conditions only for now
